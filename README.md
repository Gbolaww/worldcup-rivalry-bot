# World Cup Rivalry Bot 🏆

A Telegram bot for the truly devoted World Cup fan. Set your team and your rival, and
whenever a real goal happens in the tournament, the bot claps back with an AI-generated
hype line — and then an imagined comeback from the other side, in a completely
different voice.

Built for DEV's "Build Something Inspired by Passion" Weekend Challenge.
- **Google AI (Gemini)** generates the hype line and the rival's comeback line.
- **ElevenLabs** turns both into distinct voice notes — one voice per side of the rivalry.
- **Telegram Bot API** delivers it all in chat, in DMs or groups.

## How it works

1. `/start` — bot greets you and asks for your team
2. Send your team (e.g. `Brazil`) — bot asks for your rival
3. Send your rival (e.g. `Argentina`) — setup confirmed

   In groups, use `/team <name>` and `/rival <name>` directly — each person in a group
   tracks their own team/rival independently, even though they share the chat.

4. `/autohype` — turns on live tracking: the bot polls real World Cup 2026 results for
   both your team and your rival, and auto-fires a celebration the moment either side's
   goal total increases.
5. `/simulate` / `/simulaterival` — fires the celebration pipeline immediately (using
   your team's or rival's latest real match), without waiting on the poller. Great for
   demos, since real-world goals don't happen on command.
6. `/status` — shows your current team/rival and goals-for/goals-against tally

Every celebration is two voice notes back to back: the scoring side's hype line (in
their voice), followed by the other side's imagined clapback (in a different voice) —
generated by two separate Gemini calls and rendered with two separate ElevenLabs voice
IDs.

## Setup

1. Copy `.env.example` to `.env` and fill in your keys:
   ```
   TELEGRAM_BOT_TOKEN=...
   GEMINI_API_KEY=...
   ELEVENLABS_API_KEY=...
   ELEVENLABS_TEAM_VOICE_ID=...    # optional, defaults to ElevenLabs' "Rachel" preset
   ELEVENLABS_RIVAL_VOICE_ID=...   # optional, defaults to ElevenLabs' "Adam" preset
   ```
2. Install deps and run:
   ```
   go mod tidy
   go run main.go
   ```
   The bot uses long polling, so no public URL/webhook is needed for local dev.

   Note: Telegram only allows one active poller per bot token. Don't run this locally
   at the same time as a deployed instance is running — you'll get a 409 conflict error
   from both.

## Deploying (Railway)

- Push this repo to GitHub.
- On [Railway](https://railway.app/), create a new project from this GitHub repo.
- Set the environment variables from `.env.example` in Railway's Variables tab.
- Build command: `go build -o bot .`
- Start command: `./bot`

Railway supports true background workers, so the bot's long-polling loop runs directly
with no extra web server needed.

## Project structure

```
main.go              entrypoint — App struct, Telegram polling loop, command handling,
                     goal-celebration pipeline, background football poller
telegram/            Telegram Bot API client (long polling, send text/voice)
ai/                  Gemini text generation (hype + comeback lines) and
                     ElevenLabs text-to-speech (per-call voice ID)
football/            fetches real World Cup 2026 results from the openfootball
                     public dataset — no API key required
rivalry/             in-memory per-user state: team, rival, goal tallies, last-seen chat
```

## Notes

- User state is tracked per Telegram user ID (not per chat), so team/rival setup
  follows the person across DMs and any group they're in — multiple people in the same
  group can each root for their own team.
- User state is in-memory only (resets on restart) — fine for a hackathon demo.
- Real match data comes from a free, public, no-auth-required dataset
  ([openfootball/worldcup.json](https://github.com/openfootball/worldcup.json)) that's
  updated roughly daily rather than second-by-second — hence `/simulate` and
  `/simulaterival` as instant, demo-safe alternatives to waiting on a real goal.
